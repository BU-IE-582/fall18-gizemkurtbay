INTRODUCTION

The aim of this project is to forecast the probabilities od the 1x2 bets results of the football matches using the data of previous matches and odds. The data can be performance of the teams for the last 5 matches, odds data, day of the match, and other possible features that can give better predictions compared to predictions of other bookmakers. This can be thought as a classification problem since we could generate predictions for home win, tie win and away win, but it will be considered as regression problem due to the nature of the target class. To explain better, if a match score is 1:0 (home win), the result cannot turn into an away win (1:2) without being tie win (1:1) before. Therefore, it is decided to use multinomial logistic regression (lasso) model and stochastic gradient boosting model for the predictions in this project by finding and using the features that gives the best predictions.

Multinomial logistic regression model (lasso) is a regression analysis that performs both selection of variables and regularization to increase the accuracy of the predictions. Choosing the best lambda value that gives the smallest RPS value plays a key role in the multinomial logistic regression model (lasso), and cross validation is used in this aim. Lasso selects only a subset of the provided covariates for use in the final model rather than using all of them. Lasso penalty uses the absolute values (beta) instead of least squares. Penalized regression analysis revolves around an assumption that coefficient values around zero are more believable compared to the values far away from zero. Lasso is forcing the sum of the absolute value of the regression coefficients to be less than a fixed value, which sets the value of certain coefficients to zero, effectively choosing a simpler model that does not include those certain coefficients. Glmnet is used for penalized regression in R coding.

Stochastic gradient boosting works by constructing additive regression models by sequentially fitting a simple parameterized function, a base learner, to current pseudo-residuals by least squares at each iteration. With respect to the model values at each training data point evaluated at the current step, the pseudo-residuals give the gradient of the loss functional which is being minimized. Approximation accuracy and execution speed of stochastic gradient boosting can be improved by adding randomization into the procedure. This randomization can be obtained by taking a subsample of training data from the whole training set randomly and without replacement. The subsample taken can be used instead of the whole training dataset to fit the base learner and to update the model at each iteration. This randomization in selecting the training dataset subsample increases the robustness against overcapacity of the base learner.[1] Multinomial case can be used in stochastic gradient boosting since we have tree classes which are home, tie, away.

In the original given data there are features in the matches and odds. In matches, there are leagueId, matchId, home, away, score, date and type. LeagueId was always England Premiere League and the type is always soccer. MacthId is different for each match. Home and away represents the name of the teams playing. The score is represented as home:away. Date is turned into a number using as.factor in R coding. In odds, there are matchId, betType, oddtype, bookmaker, date, odd. MacthId is different for each match and we can use matchId to merge match and odd data since it is specific for each match. BetType is always 1x2 which represents home, tie and away. Bookmaker has the names of the different bookmakers. Date is again turned into a number using as.factor in R coding. The odds are given as numbers for each match by the bookmakers. Odds are basically the ratios of the results for 1x2 given by the bookmakers.

ANN algorithm is used for high accuracy which can cause overfitting and wasting computer resources like training time and memory. APPROACH

Two approaches are used in this project, which are multinomial logistic regression model (lasso) and stochastic gradient boosting model. The package glmnet is used for penalized regression and gbm package is used for stochastic gradient boosting model. Then RPS is used as performance metric.

The matches data is taken and preprocessed to be ready for further usage in the code. After preprocessing, the new dataset has leagueId, matchId, home, away, match date, type, weekday, month, match hour, home score, away score, match result and total score. LeagueId is always England Premiere League and the type is always soccer. MacthId is different for each match. Home and away represents the name of the teams playing in that match. Match date was first obtained as year-month-date using anydate package in R coding. Week day, month and match hour are obtained from the date using anytime package in R coding, then they are changed to factors using as.factor package in R coding . Home score, away score and total score are obtained individually as numbers. Match results are given as home, tie and away.

The dataset of odds is also preprocessed. First of all, only 1x2 bet type was considered. The bookmakers that gives odds for 1x2 bet type are taken into consideration. OddChageDateTime data is added to the dataset. Odds stayed as numbers while match dates are obtained as dates in the structure of year-month-day.

The features from matches and odds are merged together by using their matchIds. There are opening and closing odds for each matchs possible result (home, tie, away) for each bookmaker which gives odds for 1x2 bet type, and they are both used since the difference between opening and closing odds are considered as important. In the final dataset, there are matchId, leagueId, match date, weekday, moth, match hour, match result, opening and closing odds for each bookmaker which gives odds for 1x2 bet type.

The data is divided into two as train and test data and this division is done according to the dates of the matches. The date of the prediction is considered and matches with dates before the current date is considered as train data while the future dates are considered as test data. Therefore, test data has no results in the day that predictions are performed. The aim is to predict the probabilities of the results for each match accurately and RPS formulation is used for that, and RPS result is tried to be minimized.

In the aim of obtaining accurate prediction results, optimum lambda values are calculated for logistic regression by doing 10-fold cross validation with three replications and tuned accordingly based on the RPS performance trying to minimize the RPS value. Lambda is a parameter that controls the overall level (intensity) of regularization. The ?? value that minimizes the out of fold deviance is tried to be chosen, the largest ?? one standard deviation away from the minimizer lambda is chosen heuristically. [3]

Multinomial case is used in stochastic gradient boosting in R-coding since we have tree classes which are home, tie, away. The preprocessed data is changed before being used for stochastic gradient boosting. NA instances in the data set are omitted since gbm package cannot work with NA instances. For the stochastic gradient boosting model, 10-fold cross validation is done in order to prevent overfitting. Then model is obtained and used on the test data. Relative influences of features are also obtained during the R coding.

CONCLUSIONS AND FUTURE WORK

Both multinomial logistic regression model (lasso penalty) and stochastic gradient boosting are tried for each submission, and the one which is expected to give better results is uploaded on the system. While choosing the method to use, the prediction results on test data and RPS values of the previous weeks are taken into consideration. When the RPS calculation is done on the train data, the results were close to each other for multinomial logistic regression model (lasso penalty) and stochastic gradient boosting, however stochastic gradient boosting had slightly lower RPS results. However, when the RPS calculations are done on the test data, multinomial logistic regression model (lasso) was the method that generally gives better RPS values compared to stochastic gradient boosting results on the test data. The reason of multinomial logistic regression model (lasso penalty) giving lower RPS values compared to stochastic gradient boosting on test data is that, when there is low number of matches to be predicted, multinomial logistic regression model is more robust for unexpected results of surprise matches. If there were a greater number of matches to be predicted (more than 10 matches for this project), stochastic gradient boosting could have given better results, because there should be no overfitting since cross validation is done.

For a better approach, PCA could have been applied or multidimensional scaling could have been done as possible extensions, before the analyses. PCA helps to visualize the results, since most people can only understand at most tree dimensions to make conclusions on the findings. And PCA minimizes the loss of data by keeping the variation at its maximum level as much as possible while decreasing the number of dimensions to an understandable level. Using multidimensional scaling would also help for visualization of the data instead of showing numbers. In multidimensional scaling, the similar variables are closer to each other compared to the far away variables in terms of similarity measures.
